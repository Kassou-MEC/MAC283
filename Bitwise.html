Number Systems · Lecture 06

Bitwise operations

Ordinary math treats a number as a whole. Bitwise operations reach inside and work on each individual bit — the closest you get to the hardware without writing assembly. They're how a CPU flips flags, packs data, and runs fast. This lecture covers the four logic operators, the two shifts, and the everyday tricks built from them.

Motivation

Why work at the level of bits?

Because a single byte can hold eight independent yes/no facts at once, and bitwise operators let you read or change any one of them in a single, blazing-fast machine instruction.

Where you meet them

  • Flags & permissions — Unix file modes (rwx), status registers
  • Masking — pull one field out of a packed value
  • Graphics — split a color into R, G, B, A channels
  • Networking — subnet masks on IP addresses
  • Crypto & hashing — XOR is everywhere

Why it’s fast

Each operator maps straight to a logic gate the CPU already has. No loops, no branches — the hardware does all 8 (or 32, or 64) bits at the same time, in one tick of the clock. A shift can replace a whole multiply.

This is the bridge from binary numbers to the assembly instructions you'll write next unit.
Operator · AND  &

AND — 1 only if both are 1

Compare the two patterns column by column. A result bit is 1 only when both input bits are 1 — think of it as a strict gatekeeper.

Truth table

000
010
100
111

Example: 12 & 10 = 8

Used for masking: AND with a mask of 1s keeps those bits and forces every other bit to 0 — like a stencil.
Operator · OR  |

OR — 1 if either is 1

A result bit is 1 when at least one input bit is 1. It only stays 0 when both are 0 — the generous opposite of AND.

Truth table

ABA | B
000
011
101
111

Example: 12 | 10 = 14

Used for setting bits: OR with a mask turns on exactly the bits the mask has set, leaving the rest untouched.
Operator · XOR  ^

XOR — 1 if the bits differ

“Exclusive OR”: a result bit is 1 only when the two inputs are different. Same bits give 0. This little rule has surprising powers.

Truth table

ABA ^ B
000
011
101
110

Example: 12 ^ 10 = 6

Toggling: XOR with a 1 flips a bit; XOR with a 0 leaves it. So x ^ x = 0 always.
Swap with no temp: a^=b; b^=a; a^=b; exchanges two values using no extra storage.
Operator · NOT  ~

NOT — flip every bit

The only one-input operator. It inverts all bits at once: every 0 becomes 1 and every 1 becomes 0. (You met this before — it's the one's complement.)

Truth table

A~A
01
10

Example: ~12 (one byte)

Watch the sign. As an unsigned byte, ~12 = 243. As a signed (two's complement) byte it's −13 — because flipping all bits gives ~n = −n − 1. Same bits, two readings.

Moving bits

Shifts: << and >>

A shift slides every bit a fixed number of places. Bits that fall off the end are lost; zeros fill in behind. The magic is what it does to the value.

Left shift  << n

Slide left, append 0s. Each step doubles the number.

3 << 2 = 12  =  3 × 2² (×4)

Right shift  >> n

Slide right, drop the low bits. Each step halves (rounding down).

12 >> 2 = 3  =  12 ÷ 2² (÷4)

So x << n = x × 2ⁿ and x >> n = ⌊x ÷ 2ⁿ⌋. Shifts are how a CPU multiplies and divides by powers of two for free. (Right-shifting a signed negative keeps the sign bit — that's an “arithmetic” shift.)

Take it with you

One-page cheat sheet

& AND

1 if both 1. Masks / keeps bits.

| OR

1 if either 1. Sets bits on.

^ XOR

1 if different. Toggles bits.

~ NOT

Flips all bits. ~n = −n−1.

Shifts

  • x << n = x × 2ⁿ (slide left, add 0s)
  • x >> n = ⌊x ÷ 2ⁿ⌋ (slide right, drop low bits)
  • Bits pushed off the end are gone

Single-bit toolkit (bit i)

  • Check: (n >> i) & 1
  • Set: n | (1 << i) · Clear: n & ~(1 << i)
  • Toggle: n ^ (1 << i) · Odd?: n & 1
Don’t confuse them with logic. & and | work on every bit; the logical && and || work on whole true/false values. Mixing them up is a classic bug.
