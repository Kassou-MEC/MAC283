The problem

How do you store a minus sign in binary?

There are only 0s and 1s — no "−" symbol available. So engineers invented three schemes that encode "negative" using the bits themselves.

Two's Complement

💡 Understand it

Binary by itself is just a string of 0s and 1s — there's no place to put a minus sign. The trick is to agree on a rule for what a "negative" bit pattern looks like, so the hardware can tell signs apart. Three rules were tried over the years; understanding why the third one won is the goal of this whole section.

Method 1

Sign-Magnitude

The leftmost bit (MSB) becomes a sign flag: 0 = positive, 1 = negative. The remaining 7 bits hold the size (magnitude).

💡 Understand it

This is the most human-intuitive idea: just like we write a "−" in front of a number, we steal the leftmost bit to mean the sign and leave the rest as the plain magnitude. It's easy to read — but it has two real flaws we'll hit shortly: it creates two different zeros (+0 and −0), and arithmetic needs extra hardware to handle the sign separately.

Method 2

One's Complement

To make a number negative, simply flip every bit (0↔1) of its positive form.

Step 1 · start positive

+5 = 00000101

Step 2 · flip all bits

−5 = 11111010

💡 Understand it

A second idea: to negate a number, invert every bit. It's simple, and it makes subtraction easier than sign-magnitude because the hardware can mostly just add. But it still has the negative-zero problem — flipping all the bits of 00000000 gives 11111111, a second, wasteful way to write zero.

Method 3 · used by every modern CPU

Two's Complement

Flip all the bits, then add 1. That single extra step fixes everything the other methods got wrong.

1 · start with +5

00000101

2 · flip all bits

11111010

3 · add 1

11111011 = −5

💡 Why it won

The winner, used by virtually every modern processor: flip the bits and add 1. That one extra step removes negative zero entirely and — crucially — lets the CPU use the exact same adder circuit for both addition and subtraction. One piece of hardware instead of two is a massive simplification, which is why this is the standard everywhere.

💡 The payoff

Here's why two's complement wins decisively. Sign-magnitude and one's complement each waste a bit pattern on a second, meaningless "negative zero." Two's complement has exactly one zero, which means simpler comparison hardware, arithmetic that's always consistent (x − x is always 0), and one extra usable value squeezed out of the same 8 bits.

Bitwise operations

Sometimes we care about the bits themselves, not the number's value. These operations compare or flip bits one position at a time.

💡 Understand it

Sometimes a number's value doesn't matter — only its individual bits do. AND keeps a bit only if both inputs have it, OR keeps it if either does, XOR keeps it only where they differ, and NOT flips every bit. These four power permission flags, bit masks, graphics blending, and encryption — anywhere raw bits get manipulated.

Base 8

The octal system

Octal uses eight digits, 0–7, and each place is a power of 8. Because 8 = 2³, exactly three bits map to one octal digit.

Why octal?

Far easier to read than long binary strings, and historically common in early programming and embedded systems (before hex took over).

The key fact

1 octal digit = 3 bits, so converting is just grouping — no arithmetic needed.

💡 Understand it

Octal is a pure human convenience. Since 8 is 2³, every group of three binary bits corresponds to exactly one octal digit (0–7). So a long binary string collapses into a much shorter octal one with no math — you simply group the bits in threes and read each group off the table above.

Base 16

The hexadecimal system

Hex uses sixteen symbols — 0–9 then A–F (A=10, B=11, … F=15). Because 16 = 2⁴, one hex digit equals exactly four bits (a "nibble").

Memory

Addresses like 0x1A3F.

Colors

Networking

MAC addresses 00:1A:2B:3C.

Debugging

Reading raw machine code.

💡 Understand it

Hexadecimal is the programmer's favorite shorthand because 16 is 2⁴, so one hex digit equals exactly four bits. That means a whole byte (8 bits) becomes just two hex characters — compact and easy to read. The letters A–F simply stand in for the values 10 through 15, which a single decimal digit can't hold.

💡 Understand it

Hex is everywhere outside the classroom because it lines up perfectly with binary. A color like #FF5733 is simply three bytes — FF red, 57 green, 33 blue — each a number from 0–255 written in two hex digits. Memory addresses and MAC addresses use hex for the same reason: four bits per digit makes long binary values short and readable.
