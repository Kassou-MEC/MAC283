// The Vocabulary

Abbreviations Explained — Learn These First

These six abbreviations appear in every lecture this semester. They are the alphabet of computer architecture.

CPU
Central Processing Unit
The primary component that executes instructions. Every operation done with the computer is processed by the CPU.
PC
Program Counter
A register that stores the memory address of the next instruction to be executed — the CPU's bookmark.
CU
Control Unit
Directs data flow and coordinates instruction execution — the conductor of the orchestra.
ALU
Arithmetic Logic Unit
Performs mathematical calculations (add, subtract…) and logical operations (AND, OR, NOT).
RAM
Random Access Memory
Volatile memory for temporary data storage during processing. Power off → data gone.
GHz
Gigahertz
Billions of clock cycles per second — the unit that measures CPU clock speed. 3.5 GHz = 3.5 billion ticks/second.
Tip

These components work together in the Fetch–Decode–Execute cycle — we'll watch it run live in a few slides.

// The Big Idea

One Machine, Two Faces

Principle of Equivalence of Hardware & Software

Any task done by software can also be done by hardware, and any operation performed directly by hardware can be done in software.*

* assuming speed is not a concern

What this means in plain language

A computer is a machine where the physical parts (hardware) and the instructions (software) are interchangeable ways of getting work done. Hardware is fast, expensive, and frozen at manufacture. Software is flexible, cheap to change, and slower. Engineering means choosing where to draw the line.

You'll see it again in…

  • CH 4: hardwired vs microprogrammed control

// Part A · Main Components

Internal vs External Components

Internal — inside the casing

Components mounted inside the computer case, plugged directly into the motherboard:

  • CPU — the processor chip in its socket
  • RAM — memory sticks in the DIMM slots
  • Hard disk / SSD — long-term storage
  • Graphic card — renders images to the display
  • Motherboard — the board everything plugs into
External — peripheral devices

Hardware connected to the system externally, usually through USB ports:

  • Input peripherals — keyboard, mouse, scanner, microphone: they send data into the computer
  • Output peripherals — display monitor, printer, speakers, external storage: they receive results from the computer

Memory trick: Input = Into the computer. Output = Out of the computer.

Why the distinction matters

Internal components talk to the CPU over fast, short buses measured in GB/s. External peripherals go through I/O controllers and slower buses like USB — which is why I/O is the slowest part of any system, and why interrupts exist (CH 4).

CPU — The Brain and Heart of the Computer

Definition · CPU (Central Processing Unit)

The CPU is a small chip (microprocessor) that resides in the computer and carries out instructions from programs by performing basic arithmetic, logic, control, and input/output operations. Every operation done with the computer is processed by the CPU. It fits into a socket on the motherboard.

// Part A · The Processor

The Role of the CPU — The Instruction Cycle

The CPU's entire job is one loop, repeated continuously at GHz speeds. Four stages:

StageWhat happensWho does it
1 · FETCHRetrieves the next instruction from memory (RAM) at the address stored in the Program Counter (PC)PC + Memory
2 · DECODEInterprets the instruction using the Control Unit and determines the required operationsControl Unit
3 · EXECUTEPerforms the actual operation using the ALU or other componentsALU
4 · STOREWrites the result back to registers or memory, then updates the Program CounterRegisters / Memory
Note

This Fetch–Decode–Execute–Store cycle repeats continuously at GHz speeds in modern processors — a 3 GHz CPU completes roughly 3 billion cycles every second. We'll run this cycle by hand in the simulator on slide 19.

// Part A · Memory

Computer Memory — RAM vs Storage (toggle)

Where they live on the motherboard

RAM is inserted into DIMM slots (Dual Inline Memory Module) near the CPU — it snaps vertically into the slot, often installed in pairs. Storage is mounted in a drive bay (HDD/SSD, connected via SATA cable) or directly into an M.2 slot on the motherboard. RAM holds all the data that is actively being processed by the CPU.

The Motherboard — Where Everything Connects

Definition · Motherboard

The motherboard is the main printed circuit board (PCB) of the computer. It holds the CPU, RAM, storage, and expansion slots; provides power and communication paths between components; and controls data flow with onboard chipsets. It serves as the base connecting all the components of the computer.

// Part A · The Bus

What Is a Bus? — Try the Bus Width Demo

Definition · Computer Bus

A bus is a pathway used to transfer data between components — the network of electrical wires inside the computer that interconnects the hardware. Each bus is a group of wires tied together, and each wire carries 1 bit (binary 0 or 1) at a time. Buses can be internal or external. Examples: USB, PCIe, SATA, Front-Side Bus.

Bus width — why it matters

The size of a bus is its bus width — the number of wires. It determines how much data (in bits) can be transmitted at one time. A larger bus width significantly improves processor performance:

  • A 64-bit bus has 64 wires → moves 64 bits at a time
  • A 32-bit bus has 32 wires → moves 32 bits at a time

Three lanes inside the system bus
Data busTransfers the actual data between CPU, RAM, and devices
Address busCarries memory addresses from CPU to RAM or storage — the "where"
Control busSends control signals (read/write, clock) between parts — the "when & how"
Other buses you already use
PCI/PCIeConnects expansion cards (graphics, WiFi, sound)
SATAConnects storage devices — SSDs and HDDs
USBUniversal Serial Bus — connects external I/O devices
FSBFront-Side Bus — CPU↔memory hub on older boards

The observation (1965)

Gordon Moore noticed the density of transistors on a chip was doubling roughly every two years — and predicted it would continue. It held for 50+ years and drove every generation of faster, cheaper, smaller computers.

Feel the scale

// Part B · The von Neumann Model

The von Neumann Architecture — The Blueprint

Definition · von Neumann Architecture

The foundation of nearly every modern computer: one main memory holds both data AND instructions, a CPU (Control Unit + ALU) accesses that memory over a single shared system bus, with input and output devices on either side. Instructions are processed one at a time, sequentially.

The defining features
Stored inInstructions & data share the same memory
Accessed byCU fetches instructions; ALU uses the data
Bus usedOne shared system bus
Access timingOne at a time — can't fetch instruction and data simultaneously
ExampleADD A, B fetched from memory; values of A and B fetched from the same memory
The built-in traffic jam

Because everything shares one path to memory:

  • Instruction fetch delays data access
  • Data access delays instruction fetch

This is the famous von Neumann bottleneck — and the reason non-von Neumann designs exist (two slides ahead).

// Part B · Non-von Neumann Models

Breaking the Blueprint

Definition · Non-von Neumann Architecture

Computer designs that differ from the standard von Neumann model — most notably by separating data and instruction memory, or by breaking the sequential flow of execution to support parallelism, faster access, or specialized computing.

FIG 1.9 — ONE BUS vs TWO
Harvard architecture — the fix

Instructions live in instruction memory (fetched by the CU via a dedicated instruction bus); data lives in data memory (accessed by the ALU via a dedicated data bus). Both can be accessed simultaneously — no interference, no delay. Most microcontrollers and DSP chips work this way.

Parallel processing

Many CPUs/cores attack one problem together, breaking the one-instruction-at-a-time rule. Every modern multicore chip and every GPU — thousands of small processors — lives here.

Neural networks & dataflow

Computation organized like neurons, or driven by data availability instead of a program counter. The hardware behind today's AI accelerators.

Quantum computers

Qubits exploit superposition — not 0 or 1, but both at once. A fundamentally different machine, still cooled to near absolute zero.
