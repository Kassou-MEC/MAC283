The octal system

Octal is base 8 — eight digits, 0–7, and nothing else. It exists for one elegant reason: 8 = 2³, so a single octal digit packs exactly three binary bits. That makes it a compact shorthand for binary, and it's still how Unix writes file permissions. We'll cover every conversion you need.

Base 8

Digits 0–7

Place values are powers of 8: 1, 8, 64, 512…

The trick

1 digit = 3 bits

Binary ↔ octal needs no arithmetic at all.

The basics

Counting in base 8

Each column is worth 8 times the column to its right — just like decimal uses powers of 10 and binary uses powers of 2. There's no digit “8” or “9”; after 7 you carry.

The octal number 752₈ means 7×64 + 5×8 + 2×1.

Counting up

…5, 6, 7, 10, 11, …, 17, 20, …, 77, 100

After 7 the ones column resets and carries — so octal “10” is decimal 8, not ten.

Why it was invented

Early machines grouped bits in threes, and programmers needed a way to read long binary strings without going cross-eyed. Octal collapses every 3 bits into 1 digit. You'll also see it in C literals (0752) and Python (0o752).

Conversion 1

Octal → Decimal: weighted sum

Multiply each digit by its place value (a power of 8) and add. Same method you'd use to read any positional number.

Conversion 2

Decimal → Octal: divide by 8

Repeatedly divide by 8, writing down each remainder. When the quotient hits 0, read the remainders bottom-up.

Convert 490₁₀ to octal

490 ÷ 8 =61remainder 2
61 ÷ 8 =7remainder 5
7 ÷ 8 =0remainder 7

↑ read the remainders upward ↑

490₁₀ = 752₈

Why bottom-up? The first remainder you get is the ones digit (smallest place); the last is the highest place. So you stack them from the bottom.
The key insight

One octal digit = three binary bits

Because 8 = 2³, every octal digit 0–7 corresponds to exactly one group of three bits — and that mapping never changes. Learn this tiny table and binary ↔ octal becomes pure lookup.

Octal01234567
Binary 000001010011 100101110111
Compare with hex, where 16 = 2⁴ so one hex digit = 4 bits. Octal and hex are both just “binary, grouped” — octal in 3s, hex in 4s.
Conversion 3

Octal → Binary: expand each digit

Replace every octal digit with its 3-bit pattern from the table. Concatenate, and you're done — no math required.

Convert 752₈ to binary

752₈ = 111101010₂

Keep all three bits for every digit, even leading zeros (e.g. 2 → 010). You only trim leading zeros from the very front of the final result.
Conversion 4

Binary → Octal: group in threes

Starting from the right, slice the binary into groups of 3 bits (pad the left with zeros if needed). Turn each group into its octal digit.

Convert 111101010₂ to octal

111101010₂ = 752₈

Always group from the right. The rightmost bits are the ones place, so an incomplete group belongs at the left — pad it with leading zeros, never the right.

Octal in the real world

Unix permissions: chmod 755

A file's permissions are three groups — owner, group, other — each with read (4), write (2), execute (1). Add them and each group becomes a single octal digit 0–7. Toggle below and watch the number build.

755rwxr-xr-x
r = 4, w = 2, x = 1. So 7 = 4+2+1 = rwx and 5 = 4+1 = r-x. This is the same “1 octal digit = 3 bits” idea — the three bits are just labelled r, w, x.
