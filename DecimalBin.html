Binary fractions

We know how whole numbers look in binary. But how does a computer store 0.625 or 6.5? The trick is to keep the same place-value idea and let it run past the point. We start by converting decimal fractions into binary, then learn to read binary fractions back — and finish with the famous reason 0.1 + 0.2 ≠ 0.3.

The core idea

Place values keep going past the point

To the left of the point, each column is worth twice the one on its right: 8, 4, 2, 1. Keep that rule going to the right of the point and each column is worth half the previous: ½, ¼, ⅛, 1⁄16.

Same doubling rule, mirrored around the binary point. The integer columns are powers of 2; the fraction columns are powers of ½.

Each fraction column = 2⁻ⁿ: the first is 2⁻¹ = 0.5, then 2⁻² = 0.25, 2⁻³ = 0.125, 2⁻⁴ = 0.0625 …
The “binary point” is the binary version of a decimal point. Same job: it separates whole from fractional. Now let’s turn a decimal into one.
Direction 1 · the method

Decimal → binary: the multiply-by-2 trick

To convert a fraction, repeat three moves on the fraction part until it reaches 0:

1

Multiply by 2

Take the fraction and double it.

2

Record the whole part

The digit before the point (0 or 1) is your next bit.

3

Repeat on the rest

Keep only the new fraction and go again — until it hits 0.

Worked example — convert 0.625

fraction× 2= resultbitkeep
0.625×21.2510.25
0.25×20.500.5
0.5×21.010 ■ stop

Read the bits top-down: 0.625 = 0.101₂

Why it works: multiplying by 2 in binary shifts everything one place left — so the bit that lands in front of the point is exactly the next fraction digit.

Putting it together

Mixed numbers: do each side separately

A number like 6.625 has two halves. Convert the whole part the usual way, convert the fraction with multiply-by-2, then join them across the point.

Whole part: 6

Divide-by-2 (or recognize 4+2):

6 = 4 + 2 = 110₂

Fraction part: 0.625

Multiply-by-2, just like before:

0.625 = 0.101₂

Join them across the binary point:

6.625 = 110.101₂

Direction 2 · the easy way back

Reading a binary fraction

Going binary → decimal is simpler: just add the value of every column that holds a 1. This is also how you check a conversion you just did.

Example: 0.101₂

1
List the lit columns
0.5   0   0.125
=
Add them
0.5 + 0.125 = 0.625
So 0.101₂ = 0.625₁₀ — back to where slide 3 started. The middle column (¼) was a 0, so it contributes nothing.

Why this matters · floating point

The number that never ends

Watch what happens when we run multiply-by-2 on a perfectly ordinary decimal: 0.1. The “keep” column starts repeating — so the bits repeat forever.

0.1 = 0.0001100110011…₂  (the 0011 block repeats endlessly)

So 0.1 can’t be stored exactly

A computer only has room for a fixed number of bits, so it keeps a rounded version of 0.1. Tiny errors like this add up. Press the button to see the classic result:

This is why you never test if (x == 0.3) with floats — and the reason the IEEE-754 floating-point standard (next unit) exists. A fraction terminates in binary only if its denominator is a power of 2.

One-page cheat sheet

Decimal → Binary (multiply)

  • Whole part: divide-by-2 as usual
  • Fraction: ×2, write the whole digit, repeat
  • Read fraction bits top-down; stop at 0

Binary → Decimal (read)

  • Columns right of point: ½, ¼, ⅛, 1⁄16 …
  • Add the value of every column with a 1
  • Example: 0.101 = 0.5 + 0.125 = 0.625
