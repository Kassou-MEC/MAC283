The hexadecimal system

Hexadecimal is base 16 — sixteen digits: 0–9 then A–F for ten through fifteen. It exists because 16 = 2⁴, so a single hex digit packs exactly four bits — half a byte. That makes it the standard shorthand for binary: colors, memory addresses, and byte dumps are all written in hex.

The basics

Counting in base 16

Each column is worth 16 times the column to its right. We run out of normal digits after 9, so the next six values borrow letters: A=10, B=11, C=12, D=13, E=14, F=15.

The hex number 2AF₁₆ means 2×256 + 10×16 + 15×1.

Counting up

…8, 9, A, B, C, D, E, F, 10, 11, …, 1F, 20

After F the ones column resets and carries — so hex “10” is decimal 16.

Why it’s everywhere

One byte (8 bits) is exactly two hex digits, so any byte is a tidy two-character code 00–FF. That's why you see hex in #RRGGBB colors, memory addresses (0x7FFF…), MAC addresses, and hex dumps.

Conversion 2

Decimal → Hex: divide by 16

Repeatedly divide by 16, recording each remainder. Remainders 10–15 become A–F. Read the column bottom-up.

Convert 687₁₀ to hex

687 ÷ 16 =42remainder 15 → F
42 ÷ 16 =2remainder 10 → A
2 ÷ 16 =0remainder 2 → 2

↑ read the remainders upward ↑

687₁₀ = 2AF₁₆

Don't forget to translate remainders into letters: a remainder of 13 is the digit D, not “13”.
The key insight

One hex digit = four binary bits

Because 16 = 2⁴, every hex digit maps to exactly one group of four bits — a nibble. Learn this table and binary ↔ hex is pure lookup, no arithmetic.

This is the same idea as octal (1 digit = 3 bits), scaled up by one bit. Hex won because 4 bits divides a byte evenly: 1 byte = 2 nibbles = 2 hex digits.
Conversion 3

Hex → Binary: expand each digit

Replace every hex digit with its 4-bit nibble from the table, then concatenate. No math needed.

Convert 2AF₁₆ to binary

2AF₁₆ = 0010 1010 1111₂

Keep all four bits per digit, including leading zeros (2 → 0010). Trim leading zeros only from the very front of the final answer.
Conversion 4

Binary → Hex: group in fours

Starting from the right, slice the binary into nibbles of 4 bits (pad the left with zeros if needed). Convert each nibble to its hex digit.

Convert 001010101111₂ to hex

001010101111₂ = 2AF₁₆

Always group from the right, because the rightmost bits are the ones place. An incomplete nibble goes on the left and gets padded with leading zeros.

How the code is built

Each channel's decimal value (0–255) becomes one byte = two hex digits:

FF = 255 (brightest), 00 = 0 (off). So #FF0000 is pure red, #FFFFFF white, #000000 black.
